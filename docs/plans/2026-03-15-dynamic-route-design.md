# 动态区域选路与压力点参数编辑设计

## 目标

把当前“固定 3 条重点路线”的原型，升级为“用户指定起点 + 目标区域，代理人基于规则动态选路”的前端单页原型；同时允许使用者按类别开关压力点图层，并在前端临时修改部分压力点参数后重新模拟，对比热力图变化。

## 已确认约束

- 保持纯前端，无后端。
- Rhino 继续导出五个图层：`WALKABLE`、`OBSTACLE`、`NODE`、`PRESSURE`、`SEAT`。
- `NODE` 全量导出，不因新增或改名而漏掉。
- 重点代理人改为“起点任意点 + 目标区域”，不再依赖固定点对点预设。
- 路径选择采用方案 A：区域目标 + 决策节点重规划。
- 代理人每到一个 `node` 决策一次。
- 换路只针对设施选择（扶梯 / 楼梯 / 电梯），进入队列后不反悔。
- 视野半径统一为 `15m`。
- `stress rules` 中误写的 `fatigue change value` 按“压力值改变量”理解。
- 背景人流改为滑块控制，最小值 `100`，平时 `1595`，高峰 `3190`。

## 用户交互

### 1. 模型导入后的默认状态

- 仅显示灰色平面与基础底图。
- 默认不显示全部 `node` 分类标记。

### 2. 选择路线模式

- 用户点击“选择路线”后，地图显示全部已分类 `node`。
- 同一种类型的 `node` 使用同一种颜色。
- 每个 `node` 显示中文名称与类别。
- 第一次点击：选择任意起点位置（不要求必须是 `node`）。
- 第二次点击：选择“目标区域”，不是单个 `node`，而是概括性区域：
  - 出口 A / B / C / D
  - 港岛线柴湾方向
  - 荃湾线
  - 港岛线坚尼地城方向

### 3. 模拟开始

- 重点代理人从用户点击的起点出发。
- 目标只固定到“区域”，不固定到“单条路径”。
- 代理人在移动过程中到达决策 `node` 时，重新判断后续走法。

## 动态选路规则

### 决策触发

- 每到一个 `node` 决策一次。
- 不做逐帧重规划，避免抖动和来回改主意。

### 候选设施

- 只对设施选择做动态换路：
  - 扶梯
  - 楼梯
  - 电梯
- 不对目标区域本身做“反悔”。

### 路径成本

- 按规则文档采用：
  - `Total Estimated Path Cost = Walk Cost + Wait Cost + Ride Cost`
- 其中：
  - `Walk Cost = Expected Walk Time * Basal Fatigue Velocity * Crowding fatigue acceleration coefficient`
  - `Wait Cost = Expected Wait Time * (0.5 * Basal Fatigue Velocity) * Queue fatigue acceleration coefficient`
  - `Ride Cost = Expected Ride Time * Facility fatigue accumulation velocity`

### 路径选择原则

- 对当前可行候选设施分别计算路径成本。
- 若某条路径 `Total Estimated Path Cost >= Base threshold`，则不选。
- 若多条路径都可选，则选成本最低的那条。
- 若当前视野内设施都不满足阈值，继续向前寻找新的候选设施。
- 若找到最远候选仍都不满足，则取最近设施作为终点。

### 换路概率

- 只在“尚未进入队列”时生效。
- 按排队人数使用以下换路概率：
  - `< 4` 人：`0`
  - `4 - 5` 人：`22.5%`
  - `6 - 10` 人：`55%`
  - `11 - 15` 人：`87.5%`
  - `> 15` 人：`87.5%`

### 队列锁定

- 设施入口 `3m` 范围内视为排队区。
- 一旦进入排队区，代理人不再反悔换路。

## 人流与性能

### 人流控制

- 当前两档 `70 / 123` 改为滑块：
  - 最低：`100`
  - 平时：`1595`
  - 高峰：`3190`

### 渲染建议

- 当前背景代理人使用 SVG 点逐帧渲染，难以稳定承载 `3190` 人。
- 推荐改为混合渲染：
  - 背景代理人、热力图：`canvas`
  - node、重点代理人、可点击层：`SVG`

## 压力点图层与参数编辑

### 图层类别

- `noise`
- `advert`
- `signage`
- `facility`

### 图层控制

- 用户可分别开启 / 关闭各类别压力点显示。
- 图层关闭仅影响显示，不应直接丢失数据。

### 可编辑字段

- `noise`
  - `decibel`
- `advert`
  - `advert type`
  - `lux`
  - `decibel`
- `signage`
  - `feature`
  - 若后续需要，可扩展 `LCD` 的 `lux`
- `facility`
  - 本轮只查看，不开放编辑

### 编辑行为

- 点击某个压力点后，右侧显示其当前参数。
- 修改参数后，不立即持久写回文件。
- 修改仅在当前前端会话内生效。
- 用户重新点击模拟后，应生成新的热力图结果，支持前后对比。

## 本轮不做

- 不做后端存储。
- 不做保存为新 `sim.json`。
- 不做跨楼层真实三维路径模拟。
- 不做进入队列后的反悔切换。

## 风险

- `3190` 背景人流需要先解决渲染性能，否则交互会卡。
- `NODE` 自动归类依赖命名规则稳定；若命名继续变化，需要统一命名解析器。
- 现有固定路线、固定 OD 权重逻辑需要拆除或兼容，否则会和区域目标冲突。
