# WebGL Dynamic Crowd Design

## Goal

在不修改现有 UI、平面图结构、热力图公式与后端回放逻辑的前提下，把前端最吃性能的动态人流显示层替换为 `WebGL`，重点解决高人数背景人流与重点代理人在长路径回放中的卡顿、掉帧和“看起来不像真实数量”的问题。

## Confirmed Scope

- 保留当前平面图底图与界面结构。
- 保留当前热力图计算公式、后端预计算与播放数据结构。
- 保留当前旧版 `Canvas/SVG` 动态人流代码，但默认停用，作为回退路径。
- 新增 `WebGL` 动态层，仅接管：
  - 背景人流小点
  - 重点代理人

## Why This Scope

当前最重的实时渲染集中在动态点位层，而平面图与大部分 UI 属于低频或静态渲染。只替换动态层可以获得最大的性能收益，同时把错位风险、联调范围和汇报前的不确定性压到最低。

## Existing Entry Points

- 主工作区背景层：`index.html` 中 `#background-crowd-canvas`
- 主工作区动态覆盖层：`index.html` 中 `#overlay-layer`
- 播放帧刷新入口：`src/app.js` 中 `renderPlaybackFrame()`
- 旧背景点渲染：`src/app.js` 中 `renderBackgroundCrowdCanvas()`
- 旧重点代理人渲染：`src/app.js` 中 `renderOverlayLayer()`
- 背景回放插值：`src/app.js` 中 `interpolateBackgroundPlaybackAgents()`

## Target Architecture

新增一层独立的 `WebGL` 动态渲染器，和现有平面图共用同一套坐标系、视窗尺寸、缩放和平移参数。

分层关系保持为：

1. 平面图底图：继续使用现有 `SVG`
2. 动态层：切换为新的 `WebGL` 画布
3. 热力图：继续使用现有 `Canvas`
4. 其余交互性叠层：继续使用现有 `SVG`

重点代理人不再由旧 `overlay` 负责绘制，而是和背景人流一起交给 `WebGL`。这样动态点位只存在一套绘制实现，避免“背景在 GPU、重点人仍在 SVG”导致的时间轴和视觉不一致。

## Alignment Strategy

为了避免错位，`WebGL` 不自行维护独立坐标换算，而是复用当前前端已有的地图变换数据：

- 同一套 `transform`
- 同一套地图宽高
- 同一套世界坐标到屏幕坐标的映射口径

窗口变化、详情视图切换、播放推进时，`WebGL` 统一从现有状态树读取最新尺寸与变换参数并刷新。

## Fallback Strategy

- 旧 `renderBackgroundCrowdCanvas()` 不删除。
- 旧 `renderOverlayLayer()` 的静态/交互职责保留，但去掉重点代理人与背景点的默认绘制职责。
- 新增渲染模式开关，默认走 `webgl`；一旦初始化失败，自动回退到旧 `canvas/svg` 路径。

这样汇报前如果 `WebGL` 出现个别设备兼容问题，可以快速切回旧实现。

## Rendering Strategy

`WebGL` 动态层只做两类事情：

- 绘制背景点实例
- 绘制重点代理人实例

不在 `WebGL` 中重做热力图、不重做地图底图、不重做交互命中测试。颜色、半径、透明度等继续沿用当前前端已经存在的表现规则，先以稳定和性能为优先，不额外扩展视觉特效。

## Data Flow

1. 后端继续产出当前已有的背景回放场与热力图播放数据
2. 前端继续用现有逻辑推进播放时间轴
3. 每帧从当前 `scenario` 读取：
   - 可见背景人流集合
   - 重点代理人当前位置
4. 将这些数据整理为紧凑的点位缓冲并提交给 `WebGL`
5. `WebGL` 负责真正的批量绘制

## Risks

### 1. 坐标错位

通过强制复用现有 `transform` 解决，不允许并存第二套地图坐标体系。

### 2. 初始化失败或浏览器兼容问题

通过自动回退到旧渲染路径解决。

### 3. 详情视图与主工作区重复维护

第一轮只先接主工作区主舞台，详情卡片与缩略舞台继续沿用原逻辑，优先把最核心展示区域做稳。若效果稳定，再决定是否扩展到详情视图。

## Success Criteria

- 主工作区在 `700` 到 `1500` 人时明显比当前流畅。
- 背景人流与重点代理人不再由旧 `Canvas/SVG` 路径主绘制。
- 平面图、热力图和动态点位不发生视觉错位。
- `WebGL` 初始化失败时可以自动回退。
- 现有热力图公式、后端回放逻辑与 UI 结构保持不变。
