# Rules-First Simulation Refresh Design

**Date:** 2026-03-16

**Scope:** Rework the current prototype so that fatigue, cognitive load, background crowd generation, route traces, and heatmap behavior follow the latest approved rules before further UI polishing.

---

## 1. Goal

This batch prioritizes simulation credibility over surface polish. The app should stop presenting fatigue and cognitive load as simplified percentages and instead compute them from the rule tables in `Book(rules).xlsx` and `规则说明.docx`. UI changes in this batch only exist to expose those corrected values and interactions.

---

## 2. Approved Product Direction

- Keep the project as a pure frontend single-page prototype.
- Keep the current 2D workflow; 3D remains deferred.
- Preserve the existing SVG interaction layer for nodes, the focus agent, start/end markers, and clickable overlays.
- Introduce a second rendering path for dense background crowds:
  - preferred runtime path: Canvas background renderer
  - fallback path: existing SVG-style renderer retained in code for rollback
- Treat the first full run as the only authoritative computation pass for a given condition set.
- After the first pass, the scene may loop visually, but the stored heatmap/trace results remain the first-pass results unless the user changes model, route, crowd, or editable point parameters.

---

## 3. Simulation Semantics

### 3.1 Focus Agent Source

- The focus agent must be sampled from the pool of 123 healthy elderly agents.
- Sampling should respect the user’s current age / gender / BMI category filters.
- Once chosen for a run, that focus agent remains fixed for the whole first-pass calculation.

### 3.2 Background Crowd

- The background crowd slider remains adjustable.
- The midpoint marker `1595` is removed from the visible slider annotations.
- The right-side marker remains `3190` and is labeled as peak period.
- The simulation core should use the true requested background count, up to 3190.
- The floor should immediately show the requested number of background people after the slider changes.
- Background crowd points are generated by:
  1. distributing agents across OD families using current route weights,
  2. assigning each agent to a concrete route,
  3. placing each agent at a random progress position on that route,
  4. rendering them as circles with a real-world diameter of `0.5m`.

### 3.3 Looping Behavior

- First pass:
  - compute real fatigue
  - compute real cognitive load
  - deposit heat only from the focus agent as it walks
  - record the focus trace
- After the focus agent completes one full journey:
  - freeze the first-pass heatmap result
  - freeze the first-pass trace
  - reset all moving agents to their starting distribution
  - continue visual looping
- Later loops should not overwrite or accumulate a new heatmap unless the user changes inputs and explicitly reruns the simulation.

## 4. Fatigue Model

Fatigue becomes an absolute numeric value instead of a percentage.

### 4.1 Base Values

- Initial fatigue for every agent: `0`
- Base threshold by age:
  - `65-69 = 950`
  - `70-74 = 920`
  - `75-79 = 910`
  - `80-84 = 830`
  - `85+ = 670`

### 4.2 Base Fatigue Velocity

- `65-69 = 0.86`
- `70-74 = 1.00`
- `75-79 = 1.23`
- `80-84 = 1.48`
- `85+ = 1.77`

### 4.3 Environmental Multipliers

For walking fatigue accumulation, multiply base fatigue velocity by:

- crowding coefficient from density inside the `15m` perception radius
- noise coefficient from real-time environmental noise
- lighting coefficient from real-time environmental lighting
- queue coefficient when the agent is queueing for a facility

### 4.4 Facility Fatigue Velocities

When using a facility, switch to the rule-table facility velocities:

- Escalator up:
  - `65-69 = 1.7`
  - `70-74 = 2.2`
  - `75-79 = 3.1`
  - `80-84 = 4.5`
  - `85+ = 8.0`
- Escalator down:
  - `65-69 = 0.8`
  - `70-74 = 1.2`
  - `75-79 = 2.2`
  - `80-84 = 4.5`
  - `85+ = 9.8`
- Elevator:
  - `65-69 = 0.86`
  - `70-74 = 1.0`
  - `75-79 = 1.23`
  - `80-84 = 1.48`
  - `85+ = 1.77`

### 4.5 Rest Rules

- Trigger rest when `fatigue >= Base threshold`
- Search for a seat within `15m`
- If no seat is visible, continue moving slowly while searching
- Give-up time:
  - `36 seconds / multiplied environmental fatigue coefficients`
- If giving up:
  - seek a low-density open area first
  - prefer open space near the wall
- Slow walking speed while searching:
  - `50%` of normal walking speed
- Recovery:
  - sitting: `20% per minute`
  - standing: `5% per minute`
- Resume normal movement when fatigue drops below `20%` of the triggering fatigue level

## 5. Cognitive Load / Stress Model

Stress is treated as cognitive load and becomes the same scalar that drives the heatmap.

### 5.1 Base Values

- Initial stress for every agent: `0`

### 5.2 Triggered Facility Events

When relevant facilities enter the agent’s `15m` vision range, process rule-based probabilistic events once per encounter:

- Customer service center
- AI virtual service ambassador
- LCD
- Common direction Signs
- Panoramic guide map
- Hanging Signs
- Advertisement

Encounter means:
- trigger when entering the effective range,
- do not retrigger while continuously staying inside,
- allow retrigger only after leaving and re-entering later.

### 5.3 Trigger Outcomes

Apply the approved event probabilities and stress deltas from the spreadsheet:

- Customer service center:
  - fail consultation: `10%`, `+300`
  - successful consultation: `10%`, `-300`
  - not used: `80%`, `0`
- AI virtual service ambassador:
  - fail consultation: `30%`, `+200`
  - successful consultation: `20%`, `-200`
  - not used: `50%`, `0`
- LCD brief information: `30%`, `-20`
- Common direction Signs brief information: `30%`, `-20`
- Panoramic guide map detailed information: `10%`, `-40`
- Hanging Signs brief information: `40%`, `-20`
- Advertisement static ads: `30%`, `+10`
- Advertisement dynamic/flashing ads: `20%`, `+40`
- Advertisement PSA/artwork: `20%`, `-30`

### 5.4 Environmental Coefficients

For each stress delta, apply the environmental coefficient rules:

- crowd density coefficient:
  - use the `>=0` column if the delta is non-negative
  - use the `<0` column if the delta is negative
- noise coefficient:
  - use the `>=0` column if the delta is non-negative
  - use the `<0` column if the delta is negative

The resulting stress value is the quantity displayed in the right panel and deposited into the heatmap.

## 6. Environmental Field Model

### 6.1 Vision Radius

- Fixed at `15m`

### 6.2 Queue Radius

- Queue population for a facility is defined by people inside `3m` of that facility entrance

### 6.3 Noise

- Real-time environmental noise:
  - `60 dB` base lobby noise
  - plus summed decayed contribution from nearby noise sources

### 6.4 Lighting

- Real-time environmental lighting:
  - `250 lux` base lobby lighting
  - plus summed decayed contribution from nearby lighting sources

### 6.5 Decay / Influence Assumptions

The rule docs require decay-based influence but do not specify exact global formulas for all categories. This batch should use defensible fixed assumptions:

- noise and lighting:
  - radial decay with explicit finite influence radius
  - contribution becomes zero outside that radius
- signage and advertisement:
  - effective only inside the `15m` vision radius
  - event impact happens through the encounter probability model, not continuous per-frame stacking

These assumptions should be documented in code and surfaced as implementation assumptions, not hidden.

## 7. Route Selection and Wall Distance

- Agents should not hug walls.
- Minimum free walking clearance should be hardened to `2m` wherever the geometry allows.
- This affects:
  - projected walkable positions
  - route planning cost
  - local movement correction
- Exception:
  - facility/node endpoints may temporarily permit tighter approach if needed to actually reach a valid destination.

## 8. Rendering Design

### 8.1 Layer Responsibilities

- Base map: SVG
- Heatmap: Canvas
- Background crowd:
  - primary path: Canvas
  - fallback path: retained SVG-compatible path
- Focus agent, nodes, start/end badges, target highlights, trace, clickable trajectory samples: SVG overlay

### 8.2 Person Appearance

- Background people: dark gray circles, diameter `0.5m`
- Focus agent: pure black circle, same physical scale but kept visually distinguishable through layer priority

### 8.3 Nodes

- Node outline should become thinner to expose fill color
- Clicking should rely on a transparent hit area larger than the visible dot

### 8.4 Heatmap

- Remove warmup pre-coloring artifacts
- Deposit only after the focus agent truly traverses a position
- Use a rainbow palette:
  - low = purple
  - then blue, cyan, green, yellow, orange
  - high = red

---

## 9. Trace Inspection

- After the first pass completes, keep the focus trace on the map as a dark gray dashed path
- The mouse should not inspect arbitrary floor positions anymore for the focus trace review workflow
- Only sampled points on the trace are clickable
- Clicking a trace sample shows the stored snapshot at that moment:
  - fatigue value
  - cognitive load / stress value
  - crowd density
  - environmental noise
  - environmental lighting
  - threshold state

## 10. Right Panel Contract

The right panel should prioritize two core numeric fields:

- **Fatigue Value**
  - absolute number
  - red when `>= Base threshold`
- **Cognitive Load**
  - absolute number
  - same quantity as heatmap intensity

Percent formatting should be removed from these two fields.
