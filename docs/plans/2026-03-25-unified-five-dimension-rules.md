# 五维统一规则 0325 可接入版

目标：把旧 fatigue / pressure / visibility / route-choice 逻辑整理成一套可被前端运行时和后续 LLM 共用的统一规则底座。

## 统一原则

- 左侧五个能力统一为 1-5 分。
- 通用脆弱性系数统一为 `V = 1 + (3 - s) × 0.15`。
- 映射结果统一为五种负担：
  - 行动能力 -> 移动负担
  - 感知能力 -> 感知负担
  - 认知能力 -> 决策负担
  - 心理能力 -> 心理负担
  - 活力 -> 疲劳负担
- 旧逻辑保留并重命名，不推翻原先已能运行的模拟链。

## 五维计算摘要

### 1. 移动负担

- 输入：步速、局部拥挤、排队人数、最近关键设施节点
- 结果：`0-100`
- 目标：表达通行困难、设施壁垒与电梯等待压力

### 2. 感知负担

- 输入：感知半径、识别率、噪音阈值、光照舒适范围
- 结果：`0-100`
- 目标：表达能否识别导向、节点、座椅与环境提示

### 3. 决策负担

- 输入：正常导向对象、干扰对象、问题标识、关键节点、环境项
- 结果：`0-100`
- 公式：
  - `决策负担 = clamp(100 × Vc × max(0, 0.2 + 关键节点 + 干扰对象 + 问题标识 + 环境项 - 正常导向), 0, 100)`

### 4. 心理负担

- 输入：旧压力值、累计压力、可见压力、环境噪音/拥挤压力
- 结果：`0-100`
- 公式：
  - `心理负担 = clamp((旧压力值 / 400 × 100) × Vp, 0, 100)`

### 5. 疲劳负担

- 输入：旧 fatigue、fatigue threshold、感知疲劳乘数
- 结果：`0-100`
- 公式：
  - `疲劳负担 = clamp((旧 fatigue / threshold × 100) × Vv × 感知疲劳系数, 0, 100)`

## LLM 边界

- 规则引擎负责：
  - 可见对象筛选
  - 环境采样
  - 队列估算
  - 压力 / 疲劳累计
  - 五维负担计算
- LLM 负责：
  - 路径选择解释
  - 犹豫、回头、求助、避让
  - 找座位、找服务台等行为意图
  - 输出自然语言诊断与建议

## 当前限制

- 当前左侧 UI 仍是年龄/性别/BMI，不是五维滑块。
- 因此五维分值目前采用“显式传入优先，旧画像推断兜底”的方式。
- 也就是说，这一版已经是“可接入版”，但不是最终产品形态。
